# Repository Summary

## Purpose
- Template repo for orchestrated workflows; currently hosting PCB Renderer CLI scaffold (Python 3.11) that will parse/validate/render ECAD JSON inputs to SVG/PNG/PDF with strict validation rules.

## Build & Validation
- Install deps: `python -m pip install -e .[dev]`
- Tests: `python -m pytest -q`
- Lint: (planned) ruff once configured
- CI: `.github/workflows/ci.yml` (checkout -> setup-python 3.11 -> install -> pytest)

## Key Directories
- `plan_docs/` — product/architecture specs; includes `PCB Renderer CLI Specification.md`
- `src/pcb_render/` — CLI package (placeholders for cli, models, parse, validate, geometry, render)
- `tests/` — smoke test + fixture directories (`fixtures/`, `invalid_boards/`, `snapshots/`)
- `docs/debriefs/` — debrief report (`project-setup.md`)
- `docker/` — existing agent docker assets (not CLI-specific yet)

## Config & Tooling
- `pyproject.toml` — project metadata, deps, CLI entrypoint `pcb-render`
- `.gitignore` — augmented with Python artifacts (`__pycache__/`, `.pytest_cache/`, `.ruff_cache/`, `*.egg-info/`, `.venv/`)
- `README.md` — scaffold overview + quickstart
